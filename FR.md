# 需求文档

## 1. 背景

为提升开发、测试、运维等工作效率，需要提供一套 **Web 在线工具**，能够快速完成 QR 码、UUID、随机密码、加密/解密、编码/解码以及文件哈希等常用操作。全部功能以 HTTP API 和 Web 页面两种方式对外提供。

## 2. 项目目标

- **即开即用**：在浏览器中即可使用全部功能，无需登录或额外安装插件。
- **独立实现（重要限制）**：项目分为 `web`、`api` 两个部分，单独实现功能。
  - `web` 是纯前端项目，使用 **Vue 3 + Naive UI** 实现，所有工具逻辑在浏览器本地完成。
  - `api` 项目使用 **Express** 实现，要支持 **Serverless** 平台部署。
  - 二者单独实现，**Web 项目不调用 API 项目**。
- **开源友好**：代码、文档采用 **MIT** 许可证开源，方便社区二次开发。

## 3. 需求范围

| 编号   | 范围描述                                                                             |
| :----- | :----------------------------------------------------------------------------------- |
| **R1** | QR 码生成（文本、Wi‑Fi）。                                                           |
| **R2** | UUID 生成（v4、v5，支持批量 1‑100）。                                                |
| **R3** | 随机密码生成（长度 8‑256，可自定义字符集）。                                         |
| **R4** | 文本散列（MD5、SHA‑256）。                                                           |
| **R5** | 编码/解码转换（Base64、Hex、UTF‑8、Unicode、URL）。                                  |
| **R6** | 文件及文本哈希计算（MD5、SHA‑256）。                                                 |
| **R7** | Token 生成（支持前缀，长度设置）。                                                   |
| **R8** | WEB 项目提供对应响应式 UI，支持中/英双语切换，支持明暗模式切换。                     |
| **R9** | API 项目提供对应 HTTP 接口，支持 Serverless 平台部署。支持 `PASSWORD` 环境变量鉴权。 |

> ⚠️ 超出上述范围的功能（如登录、持久化存储、第三方集成等）不在本次需求内。

## 4. 功能需求

### 4.1 QR 码生成（R1）

1. **输入**：
   - 类型 `type`：`text` 或 `wifi`。
   - 当类型为 `text` 时，提供 `content`（≤ 256 字符）。
   - 当类型为 `wifi` 时，提供 `ssid`、`password`、`encryption`（WPA / WEP / nopass）。
2. **输出**：Base64 编码的 PNG 图片。
3. **错误处理**：缺少必填字段或字符超限返回统一错误码 `1000`。

### 4.2 UUID 生成（R2）

1. **输入**：
   - `version`：`v4` 或 `v5`。
   - `count`：生成数量，范围 1‑100。
   - 当 `version` 为 `v5` 时，需额外提供 `namespace` 和 `name`。
2. **输出**：UUID 字符串数组。

### 4.3 随机密码生成（R3）

1. **输入**：
   - `length`：密码长度，范围 8‑256。
   - 布尔标记：`includeUpper`、`includeLower`、`includeNumber`、`includeSymbol`。
   - `customSymbols`（可选）：用于覆盖默认特殊符号集合。
2. **约束**：
   - 至少勾选一种字符类别。
   - 若 `customSymbols` 为空且 `includeSymbol` 为 true，使用默认集合：`!@#$%^&*()_+-=[]{}|;:',.<>?`。
3. **输出**：生成的密码字符串。

### 4.4 文本散列（R4）

1. **输入**：
   - `algorithm`：`md5` 或 `sha256`。
   - `text`：待散列的文本。
2. **输出**：十六进制散列值。

### 4.5 编/码转换（R5）

1. **输入**：
   - `from`：源格式（`base64`、`hex`、`utf8`、`unicode`、`url`）。
   - `to`：目标格式（`base64`、`hex`、`utf8`、`unicode`、`url`）。
   - `text`：待转换的文本。
2. **约束**：`from` 与 `to` 不得相同。
3. **输出**：转换后的字符串。

### 4.6 文件哈希计算（R6）

1. **输入**：
   - 文件（通过 `multipart/form-data` 上传）。
   - `algorithm`：`md5` 或 `sha256`。
2. **约束**：文件大小 ≤ 100 MB，后端须采用流式读取防止内存占用过高。
3. **输出**：十六进制哈希值。

### 4.7 Token 生成（R7）

1. **输入**：
   - `prefix`（字符串，可选）：Token 前缀。
   - `length`（数值，可选）：Token 长度。
   - `count`（数值，可选）：生成数量。
2. **约束**：
   - `prefix`：默认为空，长度限制为 0‑16 字符。
   - `length`：默认为 32，取值范围 1‑256。
   - `count`：默认为 1，取值范围 1‑100。
3. **输出**：生成的 Token 字符串数组。

### 4.8 API 规范与鉴权（R9）

1. **请求方法**：均为 `POST` 请求。
2. **鉴权机制**：
   - 支持通过环境变量 `PASSWORD` 进行访问控制。
   - 若服务端未配置 `PASSWORD` 环境变量，则默认不进行鉴权。
   - 若服务端配置了 `PASSWORD` 环境变量，则客户端请求时必须携带 `X-API-PASSWORD` 请求头，且其值须与环境变量一致，否则返回未授权错误。
3. **返回结构**：
   ```json
   {
     "code": 0,          // 0 为成功，非 0 为错误码
     "msg": "成功",      // 错误时返回对应描述
     "data": { ... }     // 各接口返回的实际业务数据
   }
   ```
4. **错误码定义**：统一分配，常用错误码包括 `1000`（参数错误）、`1001`（不支持的类型）、`2000`（文件读写异常）、`4001`（未授权/密码错误）等。

### 4.9 前端 UI（R8）

1. **访问入口**：提供页面入口 `http://localhost:5173`，包含导航标签页对应各大功能。
2. **首页设计**：首页需要根据本地使用记录，以精美的卡片样式列出“最近使用的几个功能”和“最常使用的功能”。
3. **功能页布局**：每个功能页需合理布局，清晰提供 **表单输入区**、**生成/转换操作按钮**、**结果展示及一键复制区**。
4. **多语言支持 (i18n)**：UI 文案全部抽离至 i18n 资源文件。

- 支持中文与英文两种语言切换，默认语言根据系统设置自动选择。
- 每个语种文件单独抽离，方便后续拓展其他语言。

5. **主题切换**：支持一键切换明暗（Light/Dark）模式。
6. **接口文档**：如果设置了环境变量BACKEND_URL，则每个功能模块的页面，提供一个接口文档按钮，点开可以显示一个弹窗，弹窗中显示该功能对应的API接口文档。注意，只提供文档，不调用api。前端还是自己实现功能。

## 5. 非功能需求

| 编号    | 类型     | 要求                                                                                        |
| :------ | :------- | :------------------------------------------------------------------------------------------ |
| **NF1** | 性能     | 单次 API 响应时间 ≤ 300 ms（不计网络延迟）。文件哈希最大支持 100 MB，峰值内存占用 ≤ 50 MB。 |
| **NF2** | 可用性   | 系统年度可用性 ≥ 99.9%。                                                                    |
| **NF3** | 安全     | 所有接口必须通过 HTTPS；输入必须做长度、字符集校验，防止 XSS 与注入。                       |
| **NF4** | 兼容性   | 前端在 Chrome、Edge、Firefox 最新版以及移动端浏览器均能正常使用。                           |
| **NF5** | 可维护性 | 代码采用 TypeScript 编写，遵守统一的 ESLint + Prettier 规范；单元测试覆盖率 ≥ 80%。         |
| **NF6** | 国际化   | 前端 UI 必须支持中、英双语切换，所有提示信息均通过 i18n 系统加载。                          |

## 6. 约束条件

- **包管理器**：只能使用 `pnpm`（符合团队统一规范）。
- **部署环境**：Node.js 版本必须 ≥ 22。
- **开源许可证**：MIT，文档、代码均需标注版权信息。
